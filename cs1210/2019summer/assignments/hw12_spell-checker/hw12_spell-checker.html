HW 12 — Spell Checker

For this assignment you are to write a spell checker. Below are the requirements for your spell checker:

  1. The spell checker should first ask the user to enter the name of a text file to be checked; let's call this the check file. The program then opens the check file and checks for spelling mistakes. The spell checker creates an HTML file in which misspelled words are shown in red. The name of the HTML file should be the same as the name of the check file, only with .html added to the end. The spell checker has the capability of auto correcting some words. These corrected words are shown in blue in the HTML file.

  2. The text file wordsLower.txt contains a list of correctly spelled words, one per line. Your spell checker should use this file as its dictionary.

  3. The text file autoCorrect.txt contains pairs of words, one pair per line. Each pair consists of a misspelled word followed by the correct spelling. This is the file your spell checker should use for its auto correct feature.

  4. Here are some things to keep in mind about how your spell checker should work:
  • There are many words not included in wordsLower.txt, such as plurals and contractions. That's OK for this assignment. They can simply be considered misspelled.
  • When writing to the HTML file, all white space on a given line of input can be written as a single space in the HTML file.
  • The words in the dictionary are in lower case. The words you look up may sometimes be capitalized. This should be handled correctly (see the second example below).
  • Words containing digits should be ignored.
  • Punctuation is tricky. When a word is followed by a punctuation mark, like why?, the word and the punctuation mark need to be separated in order to check the spelling of the word. The colorization should only be applied to the word, not the punctuation mark. This should work for multiple punctuation marks, like Wow!?!
  1. Notes on HTML - See the HTML files shown in the examples below.
  • The beginning of the file, starting from <html> all the way down to <body> can be written before you even start checking the spelling. Notice that the name of the input file is used in the title.
  • As each word is examined it will fall into one of the following three cases:
      1. A correctly spelled word - can be written directly to the HTML file
      2. A misspelled word - written as red like this
        <span class="misspelled">word</span>
      3. A word that will be replaced - write the replacement word like this
        <span class="corrected">word</span>
  • At the end of each line a <br> tag should be written to the HTML file.
  • After all of the lines from the input file have been processed, the final </body> and </html> tags can be written.

Example

If an input file named myFile1.txt contains the following:


It was the besst of times,
it was teh worst of times


Then the following HTML file would be produced. The HTML file is named myFile1.txt.html.


   <html>
   <head>
   <title>Spell Check - myFile1.txt</title>
   <style type="text/css">
   <!--
       .misspelled {
           color: red;
       }
       .corrected {
           color: blue;
       }
   -->
   </style>
   </head>

   <body>
   It was the <span class="misspelled">besst</span> of times, <br>
   it was <span class="corrected">the</span> worst of times <br>
   </body>
   </html>
