HW 15 — Clock Class

In class we discussed encapsulation, information hiding, and the importance of developing and using a class' public interface. This assignment is intended to illustrate the importance of these concepts.

Part I

Create a class called Clock that represents a simple military style clock in which hours range from 0 to 23. This class should store the time using three integers, one each for hours, minutes, and seconds. The Clock class should have the following public interface:

Initializer

The initializer takes three parameters, one each for the hour, minute, and second. The default value for each is zero.

Accessors

There should be a separate get method for the hour, minute, and second. These should be called getHour, getMinute, and getSecond.

There should also be a method that returns all three as a tuple. This method should be called getTime.

Mutators

There should be a separate set method for the hour, minute, and second. These should be called setHour, setMinute, and setSecond.

There should also be a method that sets all three at once. This method should be called setTime. This method can take from zero to three parameters and sets only those parameters provided. The parameter names should be hour, minute, and second. For example, the instruction myClock.setTime(minute=10) should change the minutes to 10 while leaving the hours and seconds unchanged. This means that the instruction myClock.setTime() would not change the time at all.

There should be a reset method that takes no parameters and sets the hours, minuets, and seconds all to zero, which is midnight.

Special Methods

Provide the __str__ method to return a string representation of the clock. This method should return a string showing the hours, minutes, and seconds separated by colons. A leading zeros must added to single-digit values. So the starting time for our class would be displayed as 09:30:00.

Provide the __repr__ method to return a string representation of the clock. This should return teh exact same thing as the __str__ method.

Grading

You should test your Clock class using the program ClockTester.py. This program uses only the public interface of Clock class described above. This is the program that will be used to grade your Clock class. If the Clock class is working properly, then the clockTester program simply prints the message No Errors Found. If errors are found, then a message is printed for each error.

Part II

For this part of the assignment you are to change the way in which the Clock class internally stores the time. The time should now be stored using only one integer that represents the number of seconds since midnight. The public interface for the Clock class Should Not Change. The exact same ClockTester program, without being modified in any way, should produce the exact same output as before.

What to Turn In

You need to turn in both versions of your Clock class. To make things easier, you should rename your first version (from Part I) to Clock1.py and rename your second version (from Part II) to Clock2.py.
