Programming Style Guide

Docstrings

Each file, function, class, and method should include a docstring ("""). that succinctly describes the overall purpose of your code.

The docstring at the top of each file should include:

  1. Your name
  2. The name of the file
  3. The homework assignment number
  4. The date with the file was most recently modified
  5. A description of what the file does

Variable & Function Names

Names should should have meaning, not be too short, and not be too long. Variable and function names should be all lower case with words separated by underscores. You should avoid using Python keywords. Global variables should be avoided, but if you must use them, they should be all upper case with words separated by underscores, such as LINES_PER_PAGE.

Have meaning
Variable names like result or answer tell the reader nothing about the value stored in the variable. Names like average and user_id are much better. Likewise, a function name like myFunction is pretty much useless. A good rule of thumb is to have a function name describe the action performed, like get_total for example.
Not be too short
Use temperature instead of tmp or total rather than tot.
Not be too long
A name like average_individual_monthly_percent_increase is just too long.

One Exception: Loop variables are a common exception to this idea, and loop variables named i, j, j etc. are OK.

Code Organization

Your code should be well organized. Functions should be defined in one section of the program, code should be organized into functions so that blocks of code that need to be reused are contained within functions to enable that, and functions should have meaningful names.

Must Use a main() function

Each of your programs must be written using a main() function.

Comments

You must strike a balance between commenting everything and commenting nothing, both of which make the code hard to read. A good rule of thumb is to comment code that you might not understand if you looked at it in six months from now.

Maximum Line Length

No line of code should be more than 79 characters.

White Space

Whitespace (blank lines, spaces) should be used make your code easier to read. Whitespace can be use effectively to separate distinct parts of the code (e.g., space after commas in lists, blank lines between functions or between blocks of related lines within functions, etc.)

Code Reuse

As much as possible, try to avoid copying and pasting large chunks of code in your program. If you are tempted to do so, consider writing a function that contains the code you need to repeat.
