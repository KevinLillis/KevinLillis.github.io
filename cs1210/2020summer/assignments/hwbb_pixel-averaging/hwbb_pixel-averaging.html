HW bb – Pixel Averaging

Overview

In astronomy, people take pictures of objects they can see in the sky ‐ stars, planets, galaxies, Klingons, etc. Unfortunately, due to the distance of these objects, the pictures are often grainy and noisy. In order to compensate for this noise problem, photographers will sometimes take multiple pictures of the same object in quick succession and then process all of those images to create a final, noise-free image. We'll call this process Pixel Averaging.

What is Pixel Averaging?

The light from distance stars, nebula, and other objects in the sky is very dim when it finally arrives here on Earth. The Earth's atmosphere can affect image quality. Camera hardware can be quirky at times. All of these problems add up to create noisy images. Pixel Averaging is a simple algorithm to overcome these issues and create good output from not-so-good input.

The trick to making Pixel Averaging work is to capture multiple photos of a single object. Because of the issues mentioned above, each of these photos will be noisy. It turns out that this noise can be removed by averaging all of the noisy images together into one final high-quality image.

How it works: An image is basically a big two-dimensional grid. We can think of each pixel in an image having an (x, y) coordinate. Suppose we have n noisy images, all with the same height and width. Then each image i, where i is from 1 to n, has the same set of pixel coordinates. Since all the images have the same coordinates, we can go through each image, pixel-by-pixel, and access the same pixel location in each image. For each pixel location, we need to look at all of the noisy input images and calculate the average red, green, and blue values. These averages will be used for that pixel location in the clean image.
