HW 14 - Measuring Efficiency of Priority Queues

Part I — Implement Priority Queues

In this part of the assignment you are to write three classes, each of which implements the PriorityQueueOfIntegers interface. Note that this interface requires a custom exception. The three classes are:

  1. SortedArrayPQ: Stores the items in a sorted array.
  2. UnsortedArrayPQ: Stores the items in an array but makes no attempt to keep the array sorted.
  3. HeapPQ: Stores the items in a heap.

Part II — Measure Efficiency

In this part you will run experiments on your three priority queues by adding and removing n items to each, for increasing values of n, and measuring the elapsed times using the Java method System.nanoTime().

The the number of items (which we are calling n) added/removed from each priority queue needs to be large enough so differences in elapsed times are noticeable. For each different value of n, you are to test each queue on 10 different data sets of size n. Your program is to generate each data set, which should consist of n randomly generated integers ranging from 1 to 2n/3. This same set of numbers should be used to test all three priority queues for a given value of n.

The results of your experiments are to be saved in a CSV file so you can look at them in an spreadsheet and produce the required plots.

Once you have gathered the data from your runs, you are to produce two plots; one for adding items to the priority queues and one for removing items from the priority queues. The x-axis of each plot should represent the input size n and the y-axis should represent the elapsed time. Each plot should have three curves, one for each of the three priority queues. Each curve shows the average of the 10 runs for each value of n.

Note that the plots do not need to be generated by your Java program. You are using Java to generate and collect the data from the experiments. Your program is to generate a single CSV file that contains the values for your plot. You can then use a spreadsheet or other program to create the plots.

All numeric processing is to be done by your program. There should be no additional numeric manipulation required to produce the plots. The spreadsheet should only be used to produce the plots and should not be used to do any numeric computations.

What to Turn In

As with all of the programming assignments this semester, you are to create a single zip file that contains the entire project folder created by NetBeans and then submit your zip file through Blackboard.

In addition to turning in your Java program, you are also to turn in a single PDF document contining the two plots.
