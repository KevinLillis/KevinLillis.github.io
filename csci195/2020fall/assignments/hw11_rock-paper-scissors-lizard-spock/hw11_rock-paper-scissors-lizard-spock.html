HW 11 – Rock, Paper, Scissors, Lizard, Spock (RPSLS)

A variation of the traditional Rock, Paper, Scissors game was created by Sam Kass because it seems like when two players know each other well enough, 75 - 80% of games end in a tie. In this variation:

  • Scissors cut Paper
  • Paper covers Rock
  • Rock crushes Lizard
  • Lizard poisons Spock
  • Spock smashes Scissors
  • Scissors decapitate Lizard
  • Lizard eats Paper
  • Paper disproves Spock
  • Spock vaporizes Rock
  • Rock crushes Scissors

Here is a graphic of the rules rules and an explanation on YouTube.

How Your Program Should Work

The program randomly generates a number 1, 2, 3, 4, or 5 representing Rock, Paper, Scissors, Lizard, and Spock. The program then prompts you to enter a number 1, 2, 3, 4, or 5 and decides who wins, you or the computer. Below are a few sample runs. Please pay attention to the messages that are displayed. Also note how invalid entries are handled. The numbers I typed are shown in green. Your program does not need to produce any color.

Program Organization

Your program must include the following functions:

getComputer()
This function takes no arguments. It generates a random number from 1 to 5 and returns the corresponding string: Rock, Paper, Scissors, Lizard, or Spock. This function calls the getName function.
getHuman()
This function takes no arguments. It prompts the user to make a selection by showing a menu. It than reads the user input and returns the corresponding string: Rock, Paper, Scissors, Lizard, Spock or Undefined. Undefined is included because the user may enter a number other than 1, 2, 3, 4, or 5. This function calls the getName function.
getName(value)
This function takes one argument, that you can assume is an integer. The function returns one of the following strings, depending on the number given: Rock, Paper, Scissors, Lizard, Spock or Undefined.
getMessage(winner, loser)
This function takes two arguments, winner and loser. This function should only be called for games where there is a winner and a loser. It will not be called for games that end in a tie or for games whose outcome is undefined. This function returns the message to be displayed in the form winner verb loser, such as Paper covers Rock.
playGame(computer, human)
This function plays one game of PRSLS. It takes two arguments, one for the computer and one for the human. You can assume that each input will be one of the following: Rock, Paper, Scissors, Lizard, Spock or Undefined. The function determines if there is a winner, if the game is a tie, or if the game outcome is undefined. The resulting message is printed from this function. This function calls the getMessage function.
main()
The main function is very short. It calls the getComputer and getHuman functions to get the selections and then passes those selections to the playGame function. The main function does not actually read any input, generate any random numbers, or print any output.
