Homework 07 — Pig Heuristics

When playing the game of Pig you created in Homework 06, there are several different strategies you can use. For example, you can choose to continue rolling on a turn until you get ten or more points. Or you may choose to flip a coin whenever you need to decide whether or not to roll again. These different strategies are sometime called heuristics. It might be interesting to simulate playing the game of Pig using some different heuristics. That is what you are going to do for this assignment.

Assignment

  1. Modify the rollAgain method in your game of Pig so that instead of asking the user to decide whether or not to roll again, a turn continues until 10 or more points are earned for that turn. This will mean adding some parameters to the method.
  2. Run the game 10,000 times using this heuristic. Display a summary showing the percent of Poor, Average, and Excellent games produced when using this heuristic (see the sample run below). You should remove the print statements since the game will run 10,000 times.
  3. Now change the game to randomly choose whether or not to keep rolling the dice on a turn. This will also involve changing the parameters of the rollAgain method. Again run the game 10,000 times. Is this heuristic better or worse than the heuristic from Part 2 (answer using comments at the top of yoru program)?
  4. Devise and implement a heuristic that gives better results than either of the above heuristics.

    5. Note: For this assignment you are not required to keep the methods proscribed in Homework 06. You can feel free to structure your program as you wish.

Sample Run

Below is an example of how your program should look when it runs. Pay special attention to the number formatting. You will need to use System.out.printf for the formatting. 

            
            
            Results after 100000 games
            Poor      33.9%
            Average   56.4%
            Excellent  9.7%
